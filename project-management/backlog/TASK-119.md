# TASK-119 — PDF Generation from Real Quotes

## Context

The CPQ has a template editor concept (referenced in the review) with live preview showing sample data, but there is no "Generate PDF" or "Download PDF" button anywhere. Dana Chen said: "So I can design a template but I can't actually create a PDF from a real quote? What's the point? My reps need to email quotes to customers."

This task adds the ability to generate a PDF from a real quote with actual line items, pricing, and company branding. The PDF should include: company logo, quote number, date, customer info, line items table, subtotal/total, terms, and approval status.

Depends on TASK-116 (Product Catalog Picker) because PDF generation needs real product data, not free-text names.

## User Stories

**As Alex (Sales Rep)**, I want to click "Download PDF" on a completed quote and get a professional-looking document I can email to my customer, so that I don't have to recreate the quote in Google Docs.

**As Dana (VP RevOps)**, I want PDFs to include our company logo, brand colors, and legal terms, so that quotes look professional and consistent across the sales team.

**As Raj (Deal Desk Specialist)**, I want the PDF to show the approval status and approval chain, so that the customer knows the pricing has been officially approved.

## Outcomes

- "Generate PDF" button appears on the quote builder when a quote has at least one line item
- Clicking the button generates a PDF and triggers a browser download
- The PDF includes: header with company info/logo, quote details (number, date, status), customer info, line items table, subtotal/discount/total, terms section
- PDF styling follows the template editor settings (brand color, logo URL, company name, terms text)
- PDF generation takes less than 3 seconds and shows a loading state during generation

### Step 3: Create the PDF generation hook

Create `packages/twenty-front/src/modules/cpq/hooks/use-cpq-pdf-generation.ts`:

```typescript
import { pdf } from '@react-pdf/renderer';

export const useCpqPdfGeneration = () => {
  const [isGenerating, setIsGenerating] = useState(false);

  const generatePdf = useCallback(async (props: CpqQuotePdfDocumentProps) => {
    setIsGenerating(true);
    try {
      const doc = <CpqQuotePdfDocument {...props} />;
      const blob = await pdf(doc).toBlob();
      const url = URL.createObjectURL(blob);
      const link = document.createElement('a');
      link.href = url;
      link.download = `${props.quoteName || 'quote'}.pdf`;
      link.click();
      URL.revokeObjectURL(url);
    } finally {
      setIsGenerating(false);
    }
  }, []);

  return { generatePdf, isGenerating };
};
```

### Step 4: Add "Generate PDF" button to QuoteBuilderPage

Modify `packages/twenty-front/src/pages/cpq/QuoteBuilderPage.tsx`:

- Import `useCpqPdfGeneration`
- Add a "Generate PDF" button in the quote header section, next to the status badge
- Button should be disabled when no line items have been calculated (all `netTotal` are null)
- On click, gather all quote data and call `generatePdf()`
- Show loading state on button during generation

Add after the status badge div:
```tsx
<StyledButton
  onClick={() => generatePdf({
    quoteName,
    quoteDate: new Date().toISOString(),
    quoteStatus: status,
    lineItems: lineItems.map(li => ({
      productName: li.productName,
      quantity: li.quantity,
      listPrice: li.listPrice,
      discountPercent: li.discountPercent,
      netTotal: li.netTotal ?? '0',
    })),
    subtotal: subtotal.toFixed(2),
    total: subtotal.toFixed(2),
  })}
  disabled={isGenerating || lineItems.every(li => li.netTotal === null)}
>
  {isGenerating ? 'Generating...' : 'Generate PDF'}
</StyledButton>
```
